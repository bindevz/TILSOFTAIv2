6. Audit proof must become more durable than external link references alone.

Primary Sprint 14 goals:

A. Add stronger artifact-provider or signed-bundle trust
- Add at least one stronger trust path than metadata-only verification.
- Support one or both of:
  - controlled artifact-provider verification
  - signed evidence / signed bundle verification
- Where provider-backed retrieval exists, compare declared hash to recomputed artifact bytes.
- Where signed-bundle verification exists, validate signature/publisher trust according to policy.
- Do not fetch arbitrary untrusted URLs.

B. Add evidence trust tiers
- Extend evidence policy beyond a binary trusted/untrusted idea.
- Support trust levels such as:
  - metadata-verified
  - provider-verified
  - signature-verified
  - compliance-grade trusted
- Allow stricter environments to require stronger trust tiers.
- Surface trust-tier diagnostics clearly in gate results and review surfaces.

C. Add live-certification freshness policy
- Define freshness windows for required certification evidence kinds.
- Block production-like promotion when required evidence is stale.
- Support environment-specific freshness posture where appropriate.
- Return deterministic blocker codes for stale or insufficiently fresh evidence.

D. Add retention and archival policy
- Define retention classes and archival expectations for:
  - manifests
  - rollout attestations
  - evidence records
  - audit bundles/dossiers
- Add export/archival package rules and stable review outputs.
- Define stale/expired/archived proof handling semantics.

E. Harden dossier/export for durable audit review
- Improve dossier/export output so that it carries:
  - manifest identity
  - evidence trust level
  - attestation lineage
  - blocker/warning context
  - retention/archival metadata where appropriate
- Make review less dependent on mutable external references.

F. Keep emergency-path operations fully attributable under stronger trust rules
- Ensure fallback/break-glass/rollback paths still carry:
  - manifest lineage
  - incident linkage
  - stronger proof expectations when policy requires
  - after-action traceability
- Do not allow emergency paths to silently downgrade trust without explicit policy outcome.
